The gradient of the Lagrangian $L(\nu, \theta, \lambda)$ with respect to the Lagrange multiplier $\lambda$ provides the ascent direction for the dual update. Since $L$ is linear in $\lambda$, this gradient is exact (not an approximation) and equals the constraint violation: the difference between the current CVaR upper bound $H_\alpha(J^\theta(x_0), \nu)$ and the tolerance $\beta$. When $H_\alpha > \beta$ (constraint violated), the gradient is positive, pushing $\lambda$ upward to increase the penalty; when $H_\alpha < \beta$ (constraint satisfied with slack), the gradient is negative, relaxing the penalty.

$$
\nabla_\lambda L(\nu, \theta, \lambda) = \nu + \frac{1}{1 - \alpha}\,\mathbb{E}\!\left[(J^\theta(x_0) - \nu)^+\right] - \beta
$$

Where,

| $\nabla_\lambda L$                         | Gradient of the Lagrangian with respect to $\lambda$ (see [[2.0 - CVaR Lagrangian Function (Y. Chow et al., 2017)]]).                       |
| ------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| $\nu$                                       | VaR parameter (see [[1.0 - CVaR Minimization Formula (Y. Chow et al., 2017)]]).                                                             |
| $\alpha$                                    | CVaR confidence level.                                                                                                                       |
| $\mathbb{E}[(J^\theta(x_0) - \nu)^+]$     | Expected excess constraint cost above $\nu$.                                                                                                 |
| $J^\theta(x_0)$                            | Cumulative discounted constraint cost (see [[1.1 - CVaR-Constrained MDP (Y. Chow et al., 2017)]]).                                          |
| $\beta$                                     | Cost tolerance (risk budget).                                                                                                                |

The entire expression $\nu + \frac{1}{1-\alpha}\mathbb{E}[(J^\theta(x_0) - \nu)^+]$ equals $H_\alpha(J^\theta(x_0), \nu)$, so $\nabla_\lambda L = H_\alpha - \beta$. At convergence, **complementary slackness** holds: $\lambda^*(H_\alpha - \beta) = 0$. This means either $\lambda^* = 0$ (the constraint is inactive) or $H_\alpha = \beta$ (the constraint is tight). In the Monte Carlo setting, the gradient is estimated by replacing $\mathbb{E}[(J^\theta(x_0) - \nu)^+]$ with a sample average over $N$ trajectories. The $\lambda$-update operates on the **slowest timescale** in the multi-timescale stochastic approximation scheme, ensuring that $(\theta, \nu)$ have quasi-converged before $\lambda$ adjusts. The **envelope theorem** guarantees that this update performs true gradient ascent on the optimal Lagrangian value $L^*(\lambda)$.
