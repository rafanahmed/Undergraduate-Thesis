For continuous allocation — where the portfolio weight $\omega$ takes any value in $[0, 1.5]$ (including leverage) — the framework uses the **Deep Deterministic Policy Gradient (DDPG)** architecture integrated with a **Time-series Dense Encoder (TiDE)**. 

TiDE is specifically designed to handle temporal dependencies in multivariate time series more effectively than LSTMs or CNNs.
$$
\nabla_\theta J = \mathbb{E}\!\left[\nabla_a Q_\phi(s, a)\,\nabla_\theta \pi_\theta(s)\right]
$$

Where,

| $\nabla_\theta J$       | Gradient of the objective function $J$ with respect to the actor's parameters $\theta$. This is the direction in which $\theta$ should be updated to improve the policy.  |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| $\mathbb{E}[\cdot]$     | Expectation over the state distribution induced by the current policy.                                                                                                    |
| $\nabla_a Q_\phi(s, a)$ | Gradient of the critic's action-value function $Q_\phi$ with respect to the action $a$. Tells the actor *which direction* to adjust the action to increase expected value. |
| $Q_\phi$                 | Critic network parameterised by $\phi$, which estimates the expected cumulative reward for a given state-action pair. Uses a soft-updated target critic ($\tau = 0.005$).  |
| $\nabla_\theta \pi_\theta(s)$ | Gradient of the actor network's output with respect to its parameters $\theta$. Tells the optimiser how $\theta$ changes the action.                                |
| $\pi_\theta(s)$         | Actor network (the deterministic policy) that maps states $s$ directly to continuous actions $a$.                                                                         |
| $s$                     | Current state — the multivariate time-series observation (macroeconomic predictors, lagged returns).                                                                      |
| $a$                     | Action — the continuous portfolio weight $\omega \in [0, 1.5]$.                                                                                                           |

This architecture enables the agent to map complex states directly to **continuous weights** $\omega \in [0, 1.5]$.

This model-free approach is superior to model-based methods, which often suffer from the *sub-optimality* of a two-step "prediction then optimization" process. T

he chain rule structure $\nabla_a Q \cdot \nabla_\theta \pi$ is what makes end-to-end gradient flow possible: the critic provides the value landscape, and the actor navigates it. 
- The inclusion of leverage ($\omega > 1.0$) introduces a specific architectural trade-off: it increases absolute portfolio value but typically results in a lower Sharpe Ratio due to increased volatility.

The critic uses a **soft-updated target network** to stabilize training, with the update mechanism: $$\phi' \leftarrow \tau\phi + (1-\tau)\phi'$, \; \text{where} \; \tau = 0.005$$
Rather than hard-copying the critic periodically, this slowly blends the learned weights into the target network, preventing the target from changing too fast. 

The actor-critic networks use a **TiDE (Time-series Dense Encoder)** architecture: 
- A two-layer encoder flattens historical features and projects them into a latent representation using fully connected layers followed by residual blocks with ReLU activation and layer normalization. 
- The **TiDE actor** maps the encoded state to a scalar action via a final linear transformation and sigmoid activation ($\omega \in [0, 1.0]$), while the **TiDE critic** concatenates the encoder output with the actor's action and processes it through two fully connected layers with ReLU activations to output the Q-value. 
- Because the DDPG policy is deterministic, **Ornstein-Uhlenbeck (OU) noise** is added to actions during training to facilitate exploration. A **multi-step replay buffer** stores transitions $(s, a, r, s')$ to break correlations in consecutive data and allow mini-batch updates.
