The simplest approach to enforcing parameter bounds: examine the critic's gradient for each continuous parameter and set it to zero if the gradient would push the parameter beyond its allowed range. This prevents *direct* gradient-driven overflow at the boundary.

$$
\nabla p = \begin{cases} \nabla p & \text{if } p_{\min} < p < p_{\max} \\ 0 & \text{otherwise} \end{cases}
$$

Where,

| $\nabla p$   | The critic's gradient with respect to continuous parameter $p$, i.e., $\nabla_p Q(s_t, a|\theta^Q)$.  |
| ------------ | ------------------------------------------------------------------------------------------------------- |
| $p$          | Current activation (value) of the continuous parameter.                                                 |
| $p_{\min}$   | Lower bound of the parameter's valid range (e.g., $0$ for power, $-180$ for direction).                |
| $p_{\max}$   | Upper bound of the parameter's valid range (e.g., $100$ for power, $180$ for direction).                |

**Zeroing Gradients** is a baseline bounding strategy that fails in practice for two reasons. First, gradients applied to *other* network parameters can indirectly cause a bounded parameter to overflow — zeroing the gradient of $p$ does not prevent changes to shared hidden layers from affecting $p$'s output. The paper observed dash power reaching 120 despite a maximum of 100. Second, the abrupt zeroing creates discontinuities in the gradient landscape that can destabilize learning, as observed in the divergent Q-values and critic losses of one of the two zeroing-gradient agents. This failure motivates the development of **Inverting Gradients** (see [[6.0 - Inverting Gradients (M. Hausknecht & P. Stone, 2016)]]).
