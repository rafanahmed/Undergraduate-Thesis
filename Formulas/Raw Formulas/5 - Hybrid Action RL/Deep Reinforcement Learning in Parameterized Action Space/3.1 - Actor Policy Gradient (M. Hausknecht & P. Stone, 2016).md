The actor network $\mu(s|\theta^\mu)$ is updated using the critic's gradients with respect to the *action inputs*, not with respect to explicit targets. The critic's backward pass produces $\nabla_a Q(s, a|\theta^Q)$, which indicates directions in action space that increase the estimated Q-value. These action-space gradients are then backpropagated through the actor to update its parameters $\theta^\mu$.

$$
\nabla_{\theta^\mu} \mu(s) = \nabla_a Q(s, a|\theta^Q) \nabla_{\theta^\mu} \mu(s|\theta^\mu)
$$

Where,

| $\nabla_{\theta^\mu} \mu(s)$            | Gradient of the actor's output with respect to its parameters — the signal used to update $\theta^\mu$.      |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| $\nabla_a Q(s, a|\theta^Q)$             | Critic's gradient with respect to the action input (see [[3.0 - Critic Loss Function (M. Hausknecht & P. Stone, 2016)]]). Indicates how to change $a$ to increase $Q$. |
| $\nabla_{\theta^\mu} \mu(s|\theta^\mu)$ | Jacobian of the actor network: how each output (action dimension) changes with respect to each parameter $\theta^\mu$. |
| $s$                                      | Current state.                                                                                                |
| $a$                                      | Action produced by the actor $\mu(s|\theta^\mu)$.                                                            |
| $\theta^Q$                               | Critic parameters, held fixed during the actor update.                                                       |
| $\theta^\mu$                             | Actor parameters to be updated.                                                                              |

The **Actor Policy Gradient** is a chain-rule application: the critic evaluates the actor's proposed action, and the resulting gradients flow backward through both networks. Only the actor's parameters are updated; the critic's are held fixed. This gradient-passing mechanism, first used by NFQCA (Hafner & Riedmiller, 2011), is the core of the DDPG actor update. In the parameterized action space, these gradients apply to *all* output nodes — both discrete actions and continuous parameters — allowing the critic to shape the actor's selection of *which* action to take and *how* to parameterize it.
