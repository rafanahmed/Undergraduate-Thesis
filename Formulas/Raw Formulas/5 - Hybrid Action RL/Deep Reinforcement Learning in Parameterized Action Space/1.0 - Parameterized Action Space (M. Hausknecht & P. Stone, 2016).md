A **Parameterized Action Space MDP (PAMDP)** extends the standard MDP by structuring the action space as a union of discrete action types, each paired with its own continuous parameters. This formalization captures the *hybrid* nature of real-world control problems where an agent must simultaneously decide *what* to do (a categorical choice) and *how* to do it (continuous parameterization). The definition follows the notation of Masson & Konidaris (2015).

$$
A = \bigcup_{a \in A_d} (a, p_1^a, \ldots, p_{m_a}^a)
$$

Where,

| $A$                                    | The complete parameterized action space.                                                                                   |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| $A_d = \{a_1, a_2, \ldots, a_k\}$     | The finite set of $k$ discrete action types.                                                                               |
| $a$                                    | A discrete action type drawn from $A_d$.                                                                                   |
| $p_j^a$                               | The $j$-th continuous parameter associated with discrete action $a$.                                                       |
| $m_a$                                  | The number of continuous parameters for discrete action $a$.                                                               |
| $(a, p_1^a, \ldots, p_{m_a}^a)$       | An action tuple pairing the discrete selection with its continuous parameterization.                                       |
| $\bigcup$                              | Set union over all discrete action types, reflecting that the action space is the union of all possible action-parameter tuples. |

The **Parameterized Action Space** is the foundational definition for the entire PAMDP framework. It formalizes the structural assumption that actions are not purely discrete or purely continuous, but *factored* into a categorical component and a continuous component. In the Half Field Offense domain, this expands to $A = (\text{Dash}, p_1, p_2) \cup (\text{Turn}, p_3) \cup (\text{Tackle}, p_4) \cup (\text{Kick}, p_5, p_6)$, yielding four discrete action types with six total continuous parameters. The DDPG actor network mirrors this structure with two output layers — one for discrete actions, one for all continuous parameters — enabling end-to-end learning of both the action selection and parameterization.
