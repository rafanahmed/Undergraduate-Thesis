The parameterized action space is the formal definition of the *structured hybrid action manifold* that this paper's architecture is designed to navigate. It defines the complete action space as the union of discrete actions, each paired with all possible values from its own continuous parameter space. This formalization distinguishes parameterized action spaces from both flat discrete sets and single continuous spaces, and establishes the mathematical object that the hybrid actor-critic architecture decomposes into parallel sub-problems.

$$
\mathcal{A} = \bigcup_{a \in A_d} \{(a, x) \mid x \in \mathcal{X}_a\}
$$

Where,

| $\mathcal{A}$     | The complete parameterized action space. The set of all valid action tuples $(a, x)$.                  |
| ------------------ | ------------------------------------------------------------------------------------------------------ |
| $A_d$              | Finite set of discrete meta-actions: $A_d = \{a_1, a_2, \ldots, a_k\}$.                               |
| $a$                | A discrete meta-action selected from $A_d$.                                                            |
| $x$                | Continuous parameter vector selected from $\mathcal{X}_a$ for the chosen action $a$.                   |
| $\mathcal{X}_a$    | Continuous parameter space for action $a$, with $\mathcal{X}_a \subseteq \mathbb{R}^{m_a}$.           |
| $m_a$              | Dimensionality of the parameter space for action $a$. May differ across actions.                       |
| $(a, x)$           | A complete action tuple: the discrete selection $a$ paired with its continuous parameter $x$.          |

The parameterized action space imposes a **two-level hierarchy**: first select a discrete category, then specify continuous control parameters. The key structural property is that different discrete actions may have parameter spaces of *different dimensionality* ($m_a$ varies across $a$). This heterogeneity is what prevents naive approaches — an agent cannot simply output a single fixed-dimensional vector to cover all possible action-parameter combinations. The paper uses this definition to formalize **PAMDPs** (Parameterized-action Markov Decision Processes) and to motivate the parallel sub-actor decomposition. The parameterized action space is also a special case of the general hierarchical action space with one discrete layer followed by one continuous layer.
