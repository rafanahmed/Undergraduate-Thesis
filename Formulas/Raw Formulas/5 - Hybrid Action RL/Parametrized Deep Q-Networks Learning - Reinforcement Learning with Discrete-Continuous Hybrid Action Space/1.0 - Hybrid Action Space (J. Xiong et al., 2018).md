The foundational structural definition of P-DQN's problem setting. Rather than treating actions as purely discrete or purely continuous, the **hybrid action space** formalizes the hierarchical structure where an agent first selects a discrete action type $k$ from a finite set $[K]$, then specifies a continuous parameter $x_k$ drawn from a compact subset $X_k$ associated with that action type. This definition preserves the *natural factored structure* of real-world control problems and avoids the dimensionality inflation of relaxation-based approaches.

$$
A = \{(k, x_k) \mid x_k \in X_k \text{ for all } k \in [K]\}
$$

Where,

| $A$        | The complete hybrid action space — the set of all valid discrete-continuous action tuples.                            |
| ---------- | -------------------------------------------------------------------------------------------------------------------- |
| $k$        | A discrete action type, drawn from the finite set $[K] = \{1, \ldots, K\}$.                                         |
| $K$        | Total number of discrete action types.                                                                                |
| $x_k$      | The continuous parameter associated with discrete action $k$, drawn from the compact set $X_k$.                      |
| $X_k$      | The compact set in Euclidean space defining the valid range of continuous parameters for action $k$.                  |
| $(k, x_k)$ | A hybrid action tuple: the discrete selection paired with its continuous parameterization.                            |

The **Hybrid Action Space** is the structural foundation upon which the entire P-DQN framework is built. It replaces the relaxed space $\tilde{A}$ used by Hausknecht & Stone (2016), which embeds the discrete selection into additional continuous dimensions $f_k$, inflating the action space from a union of $(k, x_k)$ pairs to a full Cartesian product $F_1 \times \ldots \times F_K \times X_1 \times \ldots \times X_K$. By preserving the natural hierarchical structure, P-DQN enables the use of the $\max_k$ operator for discrete selection (tractable) while delegating continuous optimization to a separate deterministic policy network. This also enables incorporation of human demonstration data, since human players naturally produce discrete action labels without the artificial $f_k$ parameters.
