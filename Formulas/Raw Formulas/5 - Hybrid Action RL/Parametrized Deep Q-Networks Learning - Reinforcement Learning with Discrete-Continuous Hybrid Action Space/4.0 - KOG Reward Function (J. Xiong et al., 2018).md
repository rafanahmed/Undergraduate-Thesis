In King of Glory's solo mode, the true game reward (winning) is far too sparse for gradient-based learning. The paper defines a **reward shaping** function that combines time-differenced game statistics, each weighted inversely proportional to its typical scale. This provides a *dense* training signal that rewards incremental progress: gaining gold, maintaining health, killing opponents, destroying towers, and ultimately winning.

$$
\begin{aligned}
r_t &= 0.5 \times 10^{-5}(MF_t - MF_{t-1}) + 0.5(HPD_t - HPD_{t-1} + KD_t - KD_{t-1} + TD_t - TD_{t-1}) \\
&\quad + 0.001(GD_t - GD_{t-1}) + (THP_t - THP_{t-1} + BHP_t - BHP_{t-1}) + 2W
\end{aligned}
$$

Where,

| $r_t$         | Shaped immediate reward at timestep $t$.                                                                   |
| ------------- | ---------------------------------------------------------------------------------------------------------- |
| $MF_t$        | Moving forward reward: $x + y$ (hero coordinates). Encourages forward engagement.                          |
| $HPD_t$       | Health Point Difference: $\text{HeroRelativeHP}_0 - \text{HeroRelativeHP}_1$.                              |
| $KD_t$        | Kill/Death difference: $\text{Kills}_0 - \text{Kills}_1$.                                                  |
| $TD_t$        | Tower Destroyed difference: $\text{AliveTower}_0 - \text{AliveTower}_1$.                                   |
| $GD_t$        | Gold Difference: $\text{Gold}_0 - \text{Gold}_1$.                                                          |
| $THP_t$       | Tower HP Difference: $\text{TowerRelativeHP}_0 - \text{TowerRelativeHP}_1$.                                |
| $BHP_t$       | Base HP Difference: $\text{BaseRelativeHP}_0 - \text{BaseRelativeHP}_1$.                                   |
| $W$           | Winning indicator: $\text{AliveBase}_0 - \text{AliveBase}_1$. Terminal signal ($+1$ for win, $-1$ for loss).|
| Subscript $0$ | The agent's side.                                                                                           |
| Subscript $1$ | The opponent's side.                                                                                        |

The **KOG Reward Function** is a domain-specific application of reward shaping that transforms a sparse win/loss signal into a dense per-timestep reward. The time-differencing of each statistic ($MF_t - MF_{t-1}$, etc.) ensures the agent is rewarded for *improving* its relative position rather than for absolute game state, providing a gradient signal proportional to progress. The coefficients ($5 \times 10^{-6}$ for movement, $0.5$ for combat statistics, $0.001$ for gold, $1.0$ for tower HP, $2.0$ for winning) are set to normalize the different scales of each statistic. The paper notes the algorithm is not very sensitive to reasonable changes in these coefficients.
