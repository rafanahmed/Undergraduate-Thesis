The deterministic policy network is trained to maximize the total Q-value across all discrete actions, with the Q-network weights $\omega_t$ held fixed. The sum over all $K$ discrete actions ensures that the policy network learns good continuous parameters for *every* action type, not just the currently selected one. The negative sign converts maximization into a minimization objective for gradient descent.

$$
\ell_t^\Theta(\theta) = -\sum_{k=1}^{K} Q(s_t, k, x_k(s_t; \theta); \omega_t)
$$

Where,

| $\ell_t^\Theta(\theta)$                  | Policy network loss at timestep $t$ — the objective whose minimization maximizes Q across all actions.    |
| ----------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| $K$                                       | Total number of discrete action types (see [[1.0 - Hybrid Action Space (J. Xiong et al., 2018)]]).       |
| $Q(s_t, k, x_k(s_t; \theta); \omega_t)$ | Q-value at state $s_t$ for discrete action $k$ using the policy network's parameter output $x_k(s_t; \theta)$, evaluated with fixed Q-weights $\omega_t$. |
| $x_k(s_t; \theta)$                       | Policy network output: the continuous parameter for action $k$ at state $s_t$ (see [[1.1.1 - Deterministic Policy Approximation (J. Xiong et al., 2018)]]). |
| $\theta$                                  | Weights of the policy network — the parameters being optimized.                                           |
| $\omega_t$                                | Q-network weights, held fixed during the policy update at step $t$.                                       |
| $-$                                       | Negation converts the sum of Q-values into a loss to be minimized, achieving maximization of Q.           |

The **Policy Network Loss** is the dual of the Value Network Loss (see [[2.1 - Value Network Loss (J. Xiong et al., 2018)]]) — while the Q-network learns to evaluate, the policy network learns to *produce* the continuous parameters that the Q-network rates highly. The sum over all $K$ actions is a design choice: even for actions not selected at timestep $t$, the policy network receives a gradient signal pushing the continuous parameters toward Q-maximizing values. This ensures that when the discrete $\max_k$ operator switches to a different action type, the policy network already has a reasonable parameterization ready. The policy weights $\theta$ are updated with stepsize $\beta_t$, which is asymptotically larger than $\alpha_t$, placing the policy on the **faster timescale** of the two-timescale stochastic approximation.
