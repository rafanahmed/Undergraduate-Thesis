The Q-network is trained by minimizing the squared difference between its predicted action-value and the n-step target (see [[2.0 - n-step Target (J. Xiong et al., 2018)]]). This is the standard temporal difference loss adapted to the hybrid action space setting. The weights $\omega$ are updated with stepsize $\alpha_t$, which is asymptotically negligible compared to the policy stepsize $\beta_t$ to satisfy the **two-timescale** convergence condition.

$$
\ell_t^Q(\omega) = \frac{1}{2}\left[Q(s_t, k_t, x_{k_t}; \omega) - y_t\right]^2
$$

Where,

| $\ell_t^Q(\omega)$              | Value network loss at timestep $t$ — the regression objective for Q-network weights $\omega$.                    |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| $Q(s_t, k_t, x_{k_t}; \omega)$ | Q-network's predicted action-value for the observed transition $(s_t, k_t, x_{k_t})$.                           |
| $s_t$                            | State at timestep $t$.                                                                                           |
| $k_t$                            | Discrete action taken at timestep $t$.                                                                           |
| $x_{k_t}$                       | Continuous parameter used at timestep $t$.                                                                       |
| $\omega$                         | Weights of the Q-network.                                                                                        |
| $y_t$                            | [[2.0 - n-step Target (J. Xiong et al., 2018)]]. Computed using current weights $\omega_t$ and $\theta_t$.      |
| $\frac{1}{2}$                   | Scaling constant that simplifies the gradient to $Q(s_t, k_t, x_{k_t}; \omega) - y_t$.                          |

The **Value Network Loss** drives the Q-network toward the Bellman fixed point. In the experience replay setting (Algorithm 1), the gradient $\nabla_\omega \ell_t^Q$ is computed on mini-batches sampled from the replay buffer $D$, reducing the correlation between consecutive updates. In the asynchronous setting (Algorithm 2), the gradient is computed on locally collected trajectories and sent to the parameter server. The stepsize $\alpha_t$ must decay to zero while satisfying the Robbins-Monro conditions, and must be asymptotically dominated by the policy stepsize $\beta_t$ for the two-timescale argument to hold.
